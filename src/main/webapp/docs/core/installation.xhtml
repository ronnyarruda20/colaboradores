Adamantium Documentation

Installation

Adamantium provides a main template.xhtml and additional layoutmenu.xhtml, topbar.xhtml files for the base layout. These 3 files must be placed under WEB-INF folder. Provided empty-page.xhtml is a sample content page using the main template.xhtml that defines "content" as the main ui:define placeholder. Other required resources are the css, js, image and font files that are located inside resources/adamantium-layout folder, simply copy the adamantium-layout folder to your %WEB-APP-FOLDER%/resources folder so that final path would be %WEB-APP-FOLDER%/adamantium-layout/. Please refer to demo app or maven project of the demo app as the reference.

Theme is a regular PrimeFaces theme and setup is simple as defining primefaces.THEME context parameter in web.xml as "adamantium", refer to themes page for more information.

Menu Customization

Adamantium menu has two layout modes, for horizontal mode add "layout-menu-cover-top" to the "layout-menu-cover" div in layout-menu.xhtml and for vertical mode use ""layout-menu-cover-left". Note that in horizontal mode, menu always switches to left mode for responsive layout in smaller screens.

In addition, color of the top bar including menu can either be "dark" or "light". Default is light and to use dark mode add "dark" style class to the h:body component.

Core Layout

Core Layout is a set of CSS classes used to create responsive grid layouts with additional utilities. Adamantium uses Core Layout internally, however for grid layout you may also choose another utility like Grid CSS or Bootstrap Grid.

Migration Guide
2.1 to 2.1.1
  • - Replace theme jar with the new jar.
2.0.1 to 2.1
  • - Replace theme jar with the new jar.
2.0 to 2.0.1
  • - Update layout.js, adamantium-layout.css, adamantium-layout.less, core-layout.css and template.xhtml
  • - Replace theme jar with the new jar.
  • - Replace AdamantiumMenuRenderer.java with the new AdamantiumMenuRenderer.java.
1.0 to 2.0
  • - Replace theme jar with the new jar.
